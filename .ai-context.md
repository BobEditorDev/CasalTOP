# AI Context File - CasalTOP Project

## 🎯 Project Overview
**Name**: CasalTOP - Residential Financial Management System
**Purpose**: Replace spreadsheet-based financial tracking with a web application
**Philosophy**: Simple, practical, maintainable by AI agents

## 🏗️ Architecture Summary
- **Pattern**: Feature First Architecture with Domain-Driven Design
- **Layers**: Actions → Services → Repositories → Database
- **UI**: Server Components with Client Components for interactivity
- **Validation**: Zod schemas at the edge
- **Database**: PostgreSQL via Prisma ORM

## 🎯 Domain Pattern (Copy This)

Every domain follows this exact structure:

```
domain-name/
├── actions/index.ts       # Server Actions ('use server')
├── components/            # React components ('use client')
├── repositories/index.ts # Prisma access only
├── services/index.ts      # Business logic only
├── types/index.ts        # TypeScript interfaces
├── validations/index.ts  # Zod schemas
├── utils/index.ts        # Helper functions
└── index.ts             # Public exports
```

## 🔄 Data Flow Pattern

```
User Input (Component)
    ↓
Server Action (validate)
    ↓
Service (business logic)
    ↓
Repository (Prisma)
    ↓
Database
```

## 🚫 Quick Anti-Patterns

- ❌ Components importing repositories → Import services/actions
- ❌ Services using Prisma directly → Use repositories
- ❌ Missing 'use server' in actions → Always add it
- ❌ Skipping Zod validation → Always validate
- ❌ Using `any` type → Use proper types
- ❌ Forgetting revalidatePath → Always revalidate

## ✅ Quick Patterns

- ✅ Server Actions for all writes
- ✅ Zod validation at edge
- ✅ Services for business logic
- ✅ Repositories for data access
- ✅ TypeScript strict mode
- ✅ Proper error handling

## 🛠️ Common Tasks

### Add new field to existing entity:
1. Update `types/index.ts`
2. Update `validations/index.ts`
3. Update Prisma schema
4. Run `npx prisma migrate`
5. Update repository if needed
6. Update service if needed
7. Update action if needed
8. Update component if needed

### Create new domain:
1. Copy existing domain structure
2. Update types for your entity
3. Create validations
4. Implement repository
5. Implement service
6. Implement actions
7. Create components
8. Create route in app/
9. Update sidebar
10. Run validation

### Fix bug in existing code:
1. Identify the layer (action/service/repository/component)
2. Fix following the pattern
3. Run validation
4. Test the fix
5. Update tests if needed
